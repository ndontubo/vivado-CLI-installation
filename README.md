This project idea came about when I was trying to install Vivado 2017 on my Mac with an M2 chip for my digital circuits class. Obviously you can't just install Vivado for a Mac as it only runs on Linux or Windows and my Mac does not own an Intel Chip so I was kinda stuck between two options. Option A) follow a guide given by my class (https://github.com/njerig/cse100-vivado-on-silicon-mac) to install and run vivado on mac; Option B) cave in and just use the lab computers. I really wanted to have Vivado on my personal computer because that would be really convinient but in the end I couldn't get it to work properly even though for a lot of people it did work. So this is why I decided to do this project: what if you could simply run a CLI command that automates provisioning a Rosetta-accelerated Linux VM on Apple Silicon and installing/configuring AMD Vivado inside it — so the user goes from "fresh Mac" to "working Vivado + FPGA programming"
